Executive SummaryThe Problem: Traditional Applicant Tracking Systems (ATS) rely on brittle keyword matching. They often reject qualified candidates because of resume formatting issues (columns, tables) or synonyms (e.g., rejecting "Team Lead" when looking for "Manager"). Furthermore, manual screening is slow and prone to unconscious bias.The Solution: The Smart Recruiter AI is an intelligent agent that "reads" resumes like a human. It uses Docling for layout-aware parsing, OpenSearch for Hybrid Search (Keywords + Vectors), and Google Gemini for reasoning. It scores candidates based on a Job Description (JD), calculates experience mathematically, effectively removes bias via "Blind Mode," and provides explainable metrics for every decision.2. Key Technical FeaturesA. Semantic Chunking (The "Precision" Upgrade)Standard RAG splits documents arbitrarily (e.g., every 500 characters). This breaks context.Our Approach: We implemented a custom splitter that detects Markdown headers (## Experience, ## Education) generated by Docling.Benefit: When searching for "Python," the system matches the specific "Technical Skills" section rather than a vague mention in a project description 3 pages down.B. Hybrid Search (The "Recall" Upgrade)Pure vector search often misses specific IDs or rare keywords. Pure keyword search misses synonyms.Our Approach: We combine k-NN (Vector) search for concept matching ("Financial Risk" $\approx$ "Credit Modeling") with BM25 (Keyword) boosting for critical hard skills.Deduplication: The backend implements logic to aggregate multiple matching chunks and return only the single "best" version of a candidate.C. Agentic "Blind Mode" (Bias Removal)To support DE&I (Diversity, Equity, and Inclusion) initiatives, the system includes a toggle to anonymize candidates.Implementation: The prompt instructs the LLM to redact Personally Identifiable Information (PII) like Names, Addresses, and University names during the analysis phase, replacing them with "Candidate X."D. Mathematical ReasoningUnlike standard search, this tool calculates.Example: If a candidate lists "2018-2020" and "2021-Present," the Agent calculates the total years of experience to compare against the JD's requirement (e.g., "Must have 5+ years").4.
